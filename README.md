# WRPD StudyBoard — CRA 2026-2

A shared study site for you and your squad: flashcards (with hands-free
audio playback), multiple choice, true/false, a question bank anyone can
add to, a group calendar, and a simple paste-in importer for questions
generated by Claude in your regular chat. Runs entirely as static files on
GitHub Pages, with Firebase (free tier) as the shared database so everyone
sees the same data. No account or sign-up required — just type a name.

## What's in each tab

1. **Study** — Run a session as flashcards, multiple choice, or true/false.
   Filter by category and/or week, shuffle if you want. Flashcards track
   right/wrong via a "Mark correct" button; multiple choice and true/false
   score automatically as you answer (and flash green/red on each pick).
2. **Question Bank** — Add a question by hand as Standard (flashcard +
   multiple choice) or True/False. Standard questions can auto-draft 3
   wrong-answer options — edit them freely before saving. Any question can
   be flagged **gold** (guaranteed on the exam) — gold questions get a gold
   outline everywhere they show up. New questions are auto-tagged with the
   current week.
3. **Upload** — Ask Claude, in your normal chat (not this site), to turn
   your notes or a PDF into questions using the import format below, then
   paste what it gives you into this tab to preview and add it to the
   bank. No API key, no cost — it's a plain text parser.
4. **Calendar** — Shared calendar for test dates and study sessions, with
   month/week/day views.
5. **Help** — Contact info and a quick how-to.

Every question and event is stamped with whatever name the person typed in
when they opened the site.

### Weeks

Every question is automatically filed under the week it was created in:
**Week 1 is July 13–19, 2026**, Week 2 is July 20–26, and so on — each
week starts Monday. Use the Week filter on the Study tab to study just
this week's material, or everything so far.

## 1. Set up Firebase (free, ~10 minutes)

1. Go to [console.firebase.google.com](https://console.firebase.google.com)
   and create a new project (you can turn off Google Analytics, you don't need it).
2. In the project, click the **`</>`** (web app) icon to register a new web
   app. Give it any nickname. You do **not** need Firebase Hosting.
3. Firebase will show you a `firebaseConfig` object. Copy your real values into
   `js/firebase-config.js` in this project, replacing the placeholder values.
4. In the left sidebar, go to **Build > Firestore Database** and click
   **Create database**. Choose a region close to you and start in
   **production mode**.
5. Go to **Build > Authentication > Sign-in method** and enable
   **Anonymous**. This lets the site tell requests apart without anyone
   needing an account — the display name people type in is separate and
   just stored in their own browser.
6. Go to **Firestore Database > Rules** and make sure they look like:

   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /questions/{id} {
         allow read, write: if request.auth != null;
       }
       match /events/{id} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```

   Click **Publish**. Anyone who opens the deployed site can read and write
   the shared question bank and calendar — fine for a small trusted squad,
   but don't post the URL publicly.

## 2. Deploy to GitHub Pages

1. Create a new GitHub repository (public is free and simplest).
2. Upload all the files in this folder (`index.html`, `css/`, `js/`,
   `README.md`, `.nojekyll`) to the repo, preserving the folder structure.
3. In the repo, go to **Settings > Pages**.
4. Under **Build and deployment > Source**, choose **Deploy from a branch**.
5. Pick your default branch (usually `main`) and the `/ (root)` folder, then **Save**.
6. After a minute or two, GitHub will show your live URL, something like
   `https://yourusername.github.io/your-repo-name/`.
7. Share that URL with your squad. First visit, everyone picks a display name.

Any time you edit a file, just commit and push — GitHub Pages redeploys automatically.

## 3. Using the Upload tab (no API key needed)

The Upload tab doesn't call any AI service itself — it's a plain text
parser. The AI part happens in your normal chat with Claude, separately
from this site.

**Workflow:**
1. In a regular chat with Claude (claude.ai, the app, wherever), send it
   your notes or a PDF and ask it to generate flashcards, multiple choice,
   or true/false questions in the import format below.
2. Claude will give you text (or a downloadable `.txt` file) in that format.
3. Copy it, paste it into the big text box on the Upload tab, click
   **Parse Questions**.
4. Review the preview — send questions one at a time, or use **Send all to
   Question Bank** to add everything at once.

**Import format** — one question per block, separated by a blank line:

```
Q: What are the 5 principles of ethics?
A: Fidelity, Honesty, Justice, Respect, Beneficence
CATEGORY: Ethics

Q: Integrity means doing the right thing only when someone is watching.
A: False
CATEGORY: Ethics
GOLD: yes

Q: What is the speed limit exception for passing on a two-lane road?
A: 15 mph over
TYPE: mc
DISTRACTORS: 5 mph over | 10 mph over | 25 mph over
CATEGORY: Traffic Law
```

- **`Q:`** and **`A:`** are required, one line each.
- **`TYPE:`** is optional — `standard` or `mc` (flashcard + multiple
  choice) or `truefalse`. If omitted, a plain `True`/`False` answer is
  auto-detected as a true/false question; anything else defaults to
  standard.
- **`DISTRACTORS:`** is optional, up to 3 wrong answers separated by `|`.
  Only used for standard/mc questions. Leave it out and the app
  auto-generates wrong answers when you study in multiple-choice mode
  instead — DISTRACTORS just lets you (or Claude) supply better ones.
- **`CATEGORY:`** is optional free text.
- **`GOLD:`** is optional — `yes` marks it as guaranteed-on-the-exam.

If you ask Claude for this format by name ("give me questions in
WRPD StudyBoard's import format"), it should already know the shape from
this README if you share it, or you can paste the format example above
into your chat as a reference.

## 4. Changing the site's name or URL

**A. Change the in-app title/subtitle** — currently "WRPD StudyBoard" /
"CRA 2026-2". Edit these two spots in `index.html`:
- `<title>WRPD StudyBoard</title>` (browser tab title)
- `<span class="brand-title">WRPD StudyBoard</span>` and
  `<span class="brand-sub">CRA 2026-2</span>` (header)

Commit and push — no other setup needed.

**B. Change the GitHub Pages URL by renaming the repo:**
1. In the repo, go to **Settings > General**.
2. Under **Repository name**, type the new name and click **Rename**.
3. GitHub automatically updates your Pages URL and redirects the old one
   for a while — update any bookmarks/links you've shared.

**C. Use your own custom domain instead of `github.io`:**
1. Buy a domain from any registrar (Namecheap, Cloudflare, etc.) if you
   don't have one.
2. In the repo, go to **Settings > Pages > Custom domain**, and enter your
   domain (e.g. `study.yourdomain.com`).
3. At your domain registrar, add a **CNAME record** pointing your
   subdomain to `yourusername.github.io`. For a root domain, use the **A
   records** GitHub's docs list instead.
4. Wait for DNS to propagate, then check **Enforce HTTPS** in Pages
   settings once it's available.

GitHub's official guide has more detail if you hit a snag:
[docs.github.com/pages/configuring-a-custom-domain-for-your-github-pages-site](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site)

## Audio flashcards

On the Study tab, run a flashcard session and a "Start audio session"
button appears below the card. It uses the browser's built-in
text-to-speech (no API, works without a network connection once the page
is loaded) to read the question, pause for however long you set
("thinking time"), read the answer, then automatically move to the next
card — hands-free, good for studying while driving. Tap "Stop audio" any
time to take back manual control.

## Troubleshooting

- **"Couldn't connect to the shared database"** or permission errors —
  double check `js/firebase-config.js` has your real project values, and
  that the Firestore rules from section 1 are published, and that
  Anonymous auth is enabled.
- **Question bank / calendar won't load** — check the Firestore rules were
  published, and that Firestore Database itself was created.
- **Upload tab finds nothing** — check each block has both a `Q:` and
  `A:` line, and blocks are separated by a blank line. Keep each field on
  one line — the parser doesn't support multi-line questions or answers.
- **No sound / audio button does nothing** — some browsers block
  autoplay of speech until you've interacted with the page at least once;
  since starting the session is itself a click, this should be fine, but
  older browsers or ones with speech synthesis disabled will show a toast
  saying it's unsupported.
